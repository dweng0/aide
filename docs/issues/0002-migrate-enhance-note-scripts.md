## Agent Brief

**Category:** enhancement
**Summary:** Migrate note scripts from `lifeline` and enhance `note_organizer.sh` with day-boundary housekeeping.

**Current behavior:**
`note_organizer.sh`, `note_taker.sh`, and `weekly-notes.sh` live in the `lifeline` dotfiles repo. `note_organizer.sh` creates a dated note file and opens it in vim, but does not update any context files and has no day-boundary logic.

**Desired behavior:**
All three scripts live in `aide/scripts/`. `note_organizer.sh` gains three new behaviours:
1. After creating a new note file, it appends the filename to `history_index.md` (skips if the entry already exists — idempotent).
2. On a new day, before creating today's note, it checks if `daily-goals.md` has content from a previous day. If so, it prepends a `## DD-MM-YY` date header and appends the full previous content (checkboxes intact) to `goals-archive.md`, then clears `daily-goals.md`.
3. If today's note already exists, no archiving or index update occurs — the script just opens the file.

**Key interfaces:**
- `history_index.md` — append `- \`DD-MM-YY.md\`` entries under the existing `## Recent Files` section
- `goals-archive.md` — append dated sections; each section starts with `## DD-MM-YY` followed by the archived goal content
- `daily-goals.md` — cleared (not deleted) after archiving; the file must still exist

**Acceptance criteria:**
- [ ] All three scripts are present in `aide/scripts/` and executable
- [ ] Running `note_organizer.sh` for a new date appends the filename to `history_index.md`
- [ ] Running it twice on the same date does not duplicate the `history_index.md` entry
- [ ] If `daily-goals.md` has content from a previous day, it is archived with a date header before being cleared
- [ ] `goals-archive.md` retains checkbox state (`[ ]` and `[x]`) from the archived goals
- [ ] Running on the same day a second time skips archiving entirely

**Out of scope:**
- Changes to `weekly-notes.sh` beyond migrating it as-is
