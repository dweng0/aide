# #13 — install.sh: credential setup and daily-briefing scaffold

**Labels:** enhancement · wontfix

## What to build

Extend `install.sh` to set up everything needed for the morning briefing script: prompt for Gmail and calendar credentials, write them to `.notes-context/.env`, create the `.notes-context/daily-briefing/` output directory, and seed a default user-editable noise blocklist.

The `.env` file holds two values:
- `GCAL_ICAL_URL` — the secret iCal feed URL from Google Calendar Settings → "Secret address in iCal format"
- `GMAIL_APP_PASSWORD` — the 16-character App Password from Google Account → Security → App Passwords (requires 2-Step Verification)

The noise blocklist lives at `.notes-context/email-filter.md` as a plain markdown list the user can edit at any time:

```markdown
# Email noise blocklist

Senders matching any pattern below are excluded before the LLM sees your emails.
Add one pattern per line — plain text substring match against the sender address.

- noreply@
- no-reply@
- notifications@
- discord.com
- github.com
- linkedin.com
- marketing@
- newsletter@
```

If `.env` or `email-filter.md` already exist, `install.sh` skips them (idempotent).
