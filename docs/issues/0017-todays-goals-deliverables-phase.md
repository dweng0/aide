# #17 — Today's Goals: Deliverables phase

**Labels:** enhancement · done

## What to build

Add a Deliverables phase to the Today's Goals Skill that surfaces Now-horizon items from the active project's roadmap before the open drill-down.

### Project resolution

Check `~/.notes-context/projects/` for projects that have a `roadmap.md`. 

- **No projects with a roadmap**: skip this phase silently.
- **One project with a roadmap**: use it without asking.
- **Multiple projects with roadmaps**: ask "Which project are you focusing on today?" and use the chosen one.

### Surfacing Deliverables

Read the chosen project's `roadmap.md`. Extract all Deliverables in the Now horizon (non-blocked items only — blocked items are stalled and not actionable today).

Present them:

> "Your Now horizon for [project] has 2 active deliverables: 'Ship core skills' and 'Write installer README'. Are you working on any of these today?"

For each the user wants to work on, run the existing drill-down ("what does done look like for that today?", "any blockers?", "first step?") and write the result to `daily-goals.md` with the Deliverable name as the `##` heading.

### Position in session

This phase runs third, after calendar and email, before the open "anything else?" question.

### No roadmap feedback

Checking off daily subtasks derived from a Deliverable does not update `roadmap.md`. Roadmap GPS remains the sole path for marking Deliverables complete.
