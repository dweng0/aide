# #15 — morning_briefing.py: Calendar fetch via iCal feed

**Labels:** enhancement · ready-for-agent

## What to build

Add Google Calendar fetching to `scripts/morning_briefing.py`. The script fetches today's events from the iCal feed URL stored in `.notes-context/.env`, parses them with the `icalendar` library, and writes results to `.notes-context/daily-briefing/calendar.md`.

The script prints progress feedback as it runs:

```
Fetching calendar events...
Parsing events for 2026-05-15... found 3 events
Writing to ~/.notes-context/daily-briefing/calendar.md
Done.
```

### Output format

`calendar.md` contains one entry per event today, sorted by start time:

```markdown
## 09:00 — Standup
**Duration:** 30 min  
**Location:** Zoom (optional)

## 14:00 — Q2 planning sync
**Duration:** 60 min  
**Location:** Conference Room B

## 16:30 — 1:1 with Alex
**Duration:** 30 min
```

Include only events occurring on the current calendar day. All-day events are listed first without a time.

### Graceful degradation

If `.env` is missing or `GCAL_ICAL_URL` is not set, print a clear message and exit 0. The Today's Goals Skill checks for the absence of `calendar.md` and skips the calendar phase rather than failing.

## Acceptance criteria

- [ ] Script fetches the `.ics` feed from `GCAL_ICAL_URL` in `.notes-context/.env`
- [ ] Parses events using the `icalendar` library (`pip install icalendar`)

- [ ] Writes output to `.notes-context/daily-briefing/calendar.md` in the format above, sorted by start time
- [ ] All-day events appear first without a time field
- [ ] Prints progress feedback at each step with event count
- [ ] If credentials are missing, exits 0 with a descriptive message and does not write the output file
- [ ] Running twice on the same day overwrites `calendar.md` (idempotent)
