# #16 — Today's Goals: calendar and email phases

**Labels:** enhancement · done

## What to build

Update the Today's Goals Skill to add two new opening phases before the existing goal drill-down: a calendar phase and an email phase. Both read from files written by `morning_briefing.py` and surface candidates for the user to confirm.

### Phase 1 — Calendar

Read `.notes-context/daily-briefing/calendar.md`. If the file exists and has events, present them:

> "You have 3 events today: a 9am standup, a 2pm planning sync, and a 4:30pm 1:1. Do any of these need prep work as a goal?"

For each event the user wants to add, run the existing drill-down ("what does done look like for that today?") and write the result to `daily-goals.md`.

If `calendar.md` is absent or empty, skip this phase silently.

### Phase 2 — Email

Read `.notes-context/daily-briefing/emails.md`. Apply LLM judgment to each entry:

- **Always surface**: emails from senders listed in any project's `stakeholders.md`
- **Surface if action-implied**: email content implies the user needs to reply, decide, or send something
- **Skip**: everything else

Present surfaced emails one at a time. For each, ask: "Want to add a goal for this?" If yes, run the drill-down and write to `daily-goals.md`.

If `emails.md` is absent or empty, skip this phase silently.

### Existing phases unchanged

After calendar and email, the skill continues with:
- Phase 3 — Deliverables (added in #17)
- Phase 4 — Open drill-down ("anything else?")
