## What to fix

The "Morning briefing setup" section of `README.md` was written before the architecture was finalised. It now has two inaccuracies:

### 1. Calendar section describes MCP — not the iCal approach

The README currently says:

> "Calendar events are fetched live via the Google Calendar MCP integration built into Claude Code and opencode — no credentials or setup needed. The `/todays-goals` skill calls it directly during the session."

ADR 0002 (`docs/adr/0002-todays-goals-as-morning-briefing.md`) rejected MCP for calendar in favour of an iCal feed, because MCP is not available in opencode. The correct approach:

- The user copies their **Secret iCal address** from Google Calendar Settings
- This is stored as `GCAL_ICAL_URL` in `~/.notes-context/.env`
- `morning_briefing.py calendar` fetches the `.ics` feed and writes `daily-briefing/calendar.md`
- The `install.sh` prompts for `GCAL_ICAL_URL` (alongside `GMAIL_APP_PASSWORD`)

### 2. Missing `morning_briefing.py calendar` in the Scripts table

The scripts table lists `morning_briefing.py gmail` but is missing the `calendar` subcommand (being added in #15).
