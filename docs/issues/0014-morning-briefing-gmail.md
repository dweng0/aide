# #14 — morning_briefing.py: Gmail fetch via IMAP

**Labels:** enhancement · ready-for-agent

## What to build

Add Gmail fetching to `scripts/morning_briefing.py`. The script connects to Gmail via IMAP using the App Password from `.notes-context/.env`, fetches emails from the last 24 hours, applies the noise blocklist from `.notes-context/email-filter.md`, and writes pre-filtered results to `.notes-context/daily-briefing/emails.md`.

The script prints progress feedback as it runs so the user can see what is happening:

```
Connecting to Gmail...
Fetching emails from the last 24 hours... found 18
Applying noise filter... 14 filtered out, 4 remaining
Writing to ~/.notes-context/daily-briefing/emails.md
Done.
```

### Output format

`emails.md` contains one entry per surviving email:

```markdown
## From: Jane Smith <jane@example.com>
**Subject:** Q2 planning sync  
**Received:** 2026-05-15 08:32

<first ~500 characters of email body, plain text>

---
```

Truncate body to ~500 characters to keep the file LLM-readable without dumping full threads.

### Noise filtering

Load `.notes-context/email-filter.md`, extract every non-empty, non-heading line as a pattern, and exclude any email whose sender address contains that pattern (case-insensitive substring match).

### Graceful degradation

If `.env` is missing or `GMAIL_APP_PASSWORD` is not set, print a clear message and exit 0 (do not crash). The Today's Goals Skill checks for the absence of `emails.md` and skips the email phase rather than failing.

- [ ] If credentials are missing, exits 0 with a descriptive message and does not write the output file
